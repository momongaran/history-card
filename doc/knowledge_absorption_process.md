# 知識吸収プロセス (Knowledge Absorption Process)

EnergyAbsグラフに新規の知識ソース（テキスト・講義・論文等）を統合する手順。

## 背景

既存グラフは因果構造をもつ安定したネットワーク。新規知識は、既存構造に対する
「揺らぎ」として作用する。この吸収プロセス自体が、フレームワークが扱う
drift/shock → reform のインスタンスである。

吸収の目的は**既存構造の保全**と**新規情報の獲得**の両立。

## Phase 1: ソース分析 (Source Analysis)

入力テキストから因果的主張を抽出し、仮のEnergyAbsチェーンとして構造化する。

### 手順
1. テキストを通読し、因果的主張（「AがBを引き起こした」「AによりBが変化した」）を列挙
2. 各主張をノード候補として仮ID・fwType・era を付与
3. 主張間の因果関係を仮エッジとして記述
4. 結果: **ソースチェーン**（仮DAG）を得る

### 注意
- ソースの語りの順序 ≠ 因果の順序。テーマ別語りの場合、時系列を再構成する
- ソースの因果主張が正しいとは限らない。事実関係は検証可能だが、因果解釈は著者の視点
- 1テキスト = 1視点。視点のバイアスを明示的に記録する（例: 「経済構造の変容」視点）

## Phase 2: グラフマッピング (Graph Mapping)

ソースチェーンの各ノード候補を、既存グラフのノードと突き合わせる。

### 分類

| 分類 | 定義 | 処理方針 |
|------|------|----------|
| **A: 一致** | 既存ノードと同一のイベント/概念 | 既存ノードを利用。summaryの補強のみ |
| **B: 部分一致** | 同じイベントだが視点・粒度が異なる | 既存ノードのabsorbedに追加、またはsummary加筆 |
| **C: 新規概念** | 既存グラフに対応なし | 新規ノード作成 |
| **D: 横断ドリフト** | 複数の既存ノードにまたがるプロセス | 新規driftノード + 複数エッジで既存ノードに接続 |

### 判断基準: B vs C vs D
- **時間範囲が既存ノードに収まる** → B（absorbed追加）
- **時間範囲が既存ノードと一致しない独立した概念** → C（新規ノード）
- **複数の既存ノードを横断する漸進的プロセス** → D（横断ドリフト）

### Dの特殊性（最も注意を要する）
横断ドリフトは、既存グラフでは明示されていなかった「背景プロセス」を表面化させる。
これは既存構造への最大の「揺らぎ」となる。

処理方針:
1. 横断ドリフトをdriftノードとして新設
2. 既存ノードとの接続は amplifies/sustains で表現（既存の因果関係を壊さない）
3. 必要に応じて横断ドリフトを複数のdriftに分割（粒度を既存グラフに合わせる）

## Phase 3: 統合設計 (Integration Design)

マッピング結果に基づき、具体的な変更計画を設計する。

### 3.1 既存ノードへの加筆（A/B）
- summary に新たな視点を追記（例: 経済的側面）
- absorbed に内包されるイベントを追加
- **注意**: fwType, bgType, releaseType は変更しない（既存構造の保全）

### 3.2 新規ノード（C/D）
- ID: 近接する既存ノードの枝番（W025c, W038c 等）
- fwType: ソースの因果的主張に基づき判定
- cluster: 時代的に近い既存クラスタに帰属
- bgType/releaseType/patternType: 既存の分類体系に準拠

### 3.3 エッジ設計
ソースが主張する因果関係ごとに:
1. 既存エッジで表現済み → 変更なし
2. 既存エッジと矛盾 → **フラグ**（安易に変更しない）
3. 未表現 → 新規エッジ追加

### 3.4 設計原則
- **最小侵入原則**: 既存のノード・エッジの変更は最小限にする
- **エッジ優先原則**: 新規ノードより、既存ノード間のエッジ追加を優先する
- **視点保存原則**: ソースの視点は新規ノードのnoteに記録する

## Phase 4: 検証 (Validation)

### 4.1 構造的検証
- [ ] エネルギーだまりチェック: 🔴0 🟡0
- [ ] 孤立ノードなし
- [ ] 新規ノードのbgType/releaseType/patternType設定済み
- [ ] 時系列の整合性（eraの矛盾がないこと）

### 4.2 意味的検証
- [ ] 既存エッジの意味が変わっていないこと
- [ ] 新規エッジのnoteが具体的な因果説明を含むこと
- [ ] ソースの主要な因果主張がグラフ上で追跡可能なこと

## Phase 5: 品質指標 (Quality Metrics)

吸収の質を定量評価する指標。

### 5.1 網羅率 (Coverage)
ソースの因果主張のうち、グラフ上で表現されたものの割合。
```
網羅率 = 表現済み因果主張数 / 全因果主張数
```
- 目標: ≥ 80%（残り20%は粒度の問題や意見としての除外）

### 5.2 再利用率 (Reuse Rate)
ソースのノード候補のうち、既存ノードに吸収されたものの割合。
```
再利用率 = (A + B) / (A + B + C + D)
```
- 高い = 既存グラフが充実している
- 低い = 新しい視点が多い（必ずしも悪くない）

### 5.3 構造歪み度 (Structural Distortion)
既存エッジのうち、role変更が必要だったものの割合。
```
構造歪み度 = 変更エッジ数 / 既存エッジ数
```
- 目標: ≤ 5%（超えたら吸収方針を再検討）

### 5.4 接続密度 (Integration Density)
新規ノードから既存ノードへのエッジ数の平均。
```
接続密度 = 新規⇔既存エッジ数 / 新規ノード数
```
- 目標: ≥ 2.0（各新規ノードが最低2本は既存ノードと接続）

### 5.5 だまり適合率 (Reservoir Compliance)
吸収後のエネルギーだまり違反率。
- 目標: 🔴0 🟡0

## 付記: メタ的自己言及

このプロセス自体が EnergyAbs フレームワークのインスタンスである:
- 既存グラフ = institution（確立された構造）
- 新規知識 = shock/drift（外部からの揺らぎ）
- 吸収プロセス = reform（構造の再編）
- 品質指標 = 吸収後のエネルギーだまりチェックに相当

既存構造の「構造維持力」（=エッジの密度と整合性）が高いほど、新規知識の吸収は
安定するが、逆に新規概念を受け入れにくくもなる。この緊張関係の管理が本質。
