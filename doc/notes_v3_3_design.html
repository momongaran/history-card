3.6 (6) 逓増フレームワークの要否判断

逓増の概念

  • 複雑系における逓増（increasing returns）: 正のフィードバックによる自己強化
  • B/F/E（平衡→揺らぎ→発現）とは異なるダイナミクス
  • 逓増には揺らぎが不要 — 方向が定まった時点で決定論的に進む

逓増の性質

  • フレームワーク間を貫通する力学（1つのFWに閉じ込められない）
  • 方向を持ったベクトル（B/F/Eのいずれとも異なる性質）
  • ライフサイクルを持つ（始まり→強化→ネガティブフィードバック→終了）
  • 逓増自体の帰結が逓増を弱める（例: 律令の実効化→負担増→逃散→律令制形骸化）

momentum（方向性記述）の導入案

  • 各FWに「このイベントの結果、どこにエネルギーが向かうか」を記述する
  • あとからmomentumの連鎖を追うことで逓増パターンが浮かび上がる
  • 記述基準: 意図の必然性を反映した方向（分析者の後付けラベルではなく）
  • accumulationとの違い: accumulationは過去方向、momentumは未来方向

4. 課題Aの再検証: 逓増概念による実証

4.1 momentumの記述基準（確定事項）

  • momentumはE（解放）と同時にある — 解放のエネルギーの向きそのものを記述する
  • Eの「結果」や「帰結」ではない — 解放の瞬間に内在する方向性
  • 意図的か構造的帰結かを問わない — どちらも同じmomentumで記述できる
  • 1イベントに1つのmomentum — 複数の逓増パターンに寄与する場合でも、解放の方向は1つ
accumulationとの対称性: accumulation = 過去方向（何が蓄積されてこの解放に至ったか）、momentum = 未来方向（この解放がどの方向にエネルギーを放出しているか）。どちらもEを中心に据えた記述。

4.4 逓増パターンの類型

3つの実証例から、逓増パターンに類型が存在することが判明した。

類型特徴実証例
単線型方向が一貫し、各イベントが同一方向のmomentumを持つ律令国家建設（593→701、約100年、意図的）
漸進型方向は一貫だが、各イベントの当事者にその自覚がない中央統治→地方自律（939→1192、約250年、非意図的）

4.5 検証結果のまとめ

  • 課題A（臨界点なき漸進的変容）は逓増概念とmomentum記述で扱える
  • 意図/非意図の区別は記述上不要 — 同じmomentumの枠組みで記述可能
  • 逓増パターンには少なくとも3つの類型（単線型・漸進型・収斂型）がある
  • 同一イベントが複数の逓増パターンに寄与しうるが、momentum自体は1イベント1つ

5.3 F体系の拡張方針

Fの枠組み（B/F/Eの第2層）は維持したまま、Fカテゴリに「揺らぎでないもの」も含める。

維持する理由

  • 3層構造の一貫性: Fを省略するとB→Eの2層構造になるイベントが生じ、全イベント共通の比較・集計・パターン抽出が困難になる
  • 問いの共通性: 揺らぎであれ計画通りであれ、Fが答える問いは同じ — 「なぜこのイベントはこのタイミングで起きたのか」
  • 変更の最小化: 構造変更ではなくカテゴリ追加で対応できる

Fの意味の再定義

旧定義新定義
Fluctuation（揺らぎ）= 不確実性・危機F（起動因）= このイベントがこのタイミングで起きた理由
Fの頭文字の由来（Fluctuation）と実際のカテゴリ範囲にずれが生じるが、B/F/Eという記号体系として「第2層」の位置づけで機能させる。

Fカテゴリの拡張案

カテゴリ性質例
F-UNC（既存）不確実性・危機型王位空位、災害発生、外圧急変、経済不安、権力空白
F-PRG（新規・仮称）上位プログラムの実行条件成立律令編纂条件の整備、鎖国政策の段階的実行、明治維新の個別政策
F-PRGのサブカテゴリ設計、および他に必要なFカテゴリがないかは、課題(3)のF体系再設計で検討する。

  • 課題B（上位プログラムの一部）は、上位のEが下位のFとして作用する構造

6. 全体俯瞰: B/F/E記述の視点ルールと再設計方針

6.1 B×FがEを決定する

F単独ではEの性質を決定しない。BとFの組み合わせがEを決定する。

例: 崇峻暗殺と推古即位は同じF（権力空白 F-UNC-05）だが、Bが異なるためEの性質が異なる。

6.2 記述の視点ルール — 3案の比較

B/F/Eの記述において「誰の視点で書くか」が問題になる。崇峻暗殺の例では、現行は天皇視点だが、Eの実行者（蘇我馬子）の視点で書くほうが自然である。

案1: Eの実行者視点で統一（B/F/E全て同一視点）

  • シンプルで一貫性がある
  • リスク: 同じ構造的背景が実行者によって異なる表現になり、Bのパターンマッチングが困難になる
  • 例: 「蘇我氏の権力独占」が、馬子視点では「自らの権力基盤への脅威」、中大兄皇子視点では「蘇我氏の専横による統治の形骸化」

案2: Bは客観記述、F/Eは実行者視点（有力）

  • Bは構造的背景であり、特定の誰かの視点に依存しない客観的記述が本来の性質に合う
  • Fは「この人がこのタイミングで動いた理由」であり実行者視点が自然
  • Eは「この人が放出したエネルギーの方向」であり実行者視点が自然
  • Bのパターンマッチングが維持される

案3: Bに客観記述と実行者視点記述の両方を持つ

  • 客観記述: パターンマッチング用（例: 「蘇我氏の権力独占構造」）
  • 実行者視点記述: B×F→Eの説明用（例: 「自らの権力基盤が脅かされつつあった」）
  • 情報量は最も豊富だが、データ構造が複雑化する
現時点では案2を第一候補とする。Bは客観、F/Eは実行者視点。試行の上、パターンマッチング等に支障が生じた場合は案3や別案を検討する。
