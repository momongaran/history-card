myHistory 仕様 v2.2（Freeze継続 / 継承リンク対応）

実装確定版 — 2026-02-22

1. 前提

  • v2.1 Freeze を壊さない。
  • UI機能追加はしない。既存表示ロジックは変更しない。
  • レイヤー構造 C / F / P / R はそのまま維持。
  • F（揺らぎ）は必須1（v2.1と同一）。

2. v2.2 追加事項

2.1 frameworkView.meta の拡張許容

以下のキーを読み込み可能にする。未存在でもエラーにしない。UI表示は不要。

キー型説明
meta.backgroundInheritanceArray背景要素の継承元リスト
meta.backgroundDerivationArray背景要素の派生先リスト

2.2 継承リンク（別ファイル）

inheritance_links_v2_2.json を別ファイルとして読み込み、内部保持のみ行う。

{
  "version": "2.2",
  "inheritanceLinks": [
    {
      "fromEventId": "EV_prev",
      "toEventId": "EV_next",
      "backgroundRefs": ["EL_prev_C_01"],
      "reason": "任意メモ"
    }
  ]
}

2.3 backgroundInheritance（meta内）形式

{
  "backgroundInheritance": [
    {
      "fromEventId": "EV_prev",
      "backgroundRefs": ["EL_prev_C_01"],
      "reason": "任意メモ"
    }
  ]
}

3. バリデーション

コンソール警告のみ（画面表示なし）。

  • fromEventId → toEventId の時系列逆転を検知。
  • backgroundRefs が存在する elementId を参照しているか確認。

4. やらないこと

  • グローバルレーンは作らない。
  • 背景辞書は作らない（核と具体を分離）。
  • レイヤー追加はしない。
  • 解放レイヤーを新設しない。
  • 結果の意味変更はしない。
  • 事後評価をカテゴリ基準に使わない。

5. 原則

  • 背景同一性は操作的に柔軟。
  • UI互換を壊さない。
  • v2.1と同じ見た目で動作する。
  • 継承リンクを将来使える構造だけを準備する。
