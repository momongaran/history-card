## 1. B（Background / 背景）

### 1.1 定義（拡張版）

イベント発生時に存在していた構造的条件。以下のいずれか（OR）：

- **エネルギー型:** 矛盾・恐怖・対立・圧力といった構造的勾配を持つ条件。Bの基本形。
- **土台型:** それ自体はエネルギーを持たないが、エネルギー型Bが生じる前提となる構造的事実。`origin` フィールドを持つ場合にのみ使用する。

### 1.2 視点ルール

- **客観記述:** 当時の人間が認識できた条件として記述する。
- **逆算禁止:** イベントの結果から逆算して背景を構成してはならない。「この結果が起きたということは、こういう背景があったはずだ」という推論は不可。

### 1.3 記述ルール

| フィールド | ルール |
|---|---|
| `label` | 具体的な事実記述。**状態文**で書く（「〜している」「〜にある」「〜が進んでいる」等）。**現在形を正とし、過去形（「〜していた」「〜にあった」等）は不可。** イベント発生時点で既に存在していた条件であることを文体で示す。 |
| `normalizedLabel` | labelの要約（短縮形）。 |
| `category` / `subCategory` | taxonomy定義に準拠。 |

> **文体の根拠:** Bは「イベント発生時に既に存在していた構造」であり、時系列上で「完了した出来事」ではない。「〜している」という進行・状態形がこの性質を正確に表現する。

### 1.4 個数

1イベントあたり **2〜5個**（平均3個）。

### 1.5 カテゴリ一覧（11種）

| コード | 名称 | サブカテゴリ例 |
|---|---|---|
| B-PWR | 権力配置 | 中央集権構造、豪族間均衡、分権的配置、実権と名目の分離 |
| B-LEG | 正統性基盤 | 血統的正統性、王統継承安定性、宗教的正統化、制度的正当化 |
| B-INS | 制度枠組み | 官制構造、税制財政制度、法制度枠組み、行政運用基盤、軍事制度体系 |
| B-SOC | 社会構造 | 身分階層構造、地域社会構造、経済階層分化、共同体結束構造 |
| B-RES | 資源基盤 | 財政資源、軍事資源、人的資源、物流供給基盤 |
| B-EXT | 対外環境 | 外交圧力、軍事脅威、文化流入、経済交易圏 |
| B-MIL | 軍事基盤 | 軍事組織構造、軍事技術水準、動員体制、防衛拠点配置 |

## 2. F（Fluctuation / 揺らぎ）

### 2.1 定義（拡張版）

以下のいずれか（OR）に該当する具体的事象：

- **A) 意図的行動型:** ある主体が意図を持って行った行動。**実行者視点**で記述する（「この人がこのタイミングで動いた理由」）。
- **B) 偶発事象型:** 意図的な主体を持たない事象（災害・崩御・外的衝撃など）。**影響を受ける側の視点**で記述する。

### 2.2 3条件（共通）

Fは以下の3条件を**すべて**満たさなければならない：

1. **具体的事象である** — 抽象的傾向や長期的トレンドではない。
2. **分岐を引き起こす** — Fなしには即座に同じ帰結に至らない（counterfactual test）。
3. **単独では構造を説明できない** — Bなしには事象は吸収される（Fだけでは結果が決まらない）。

### 2.3 記述ルール

| フィールド | ルール |
|---|---|
| `label` | **事象文**で書く（「〜した」「〜が発生した」「〜を察知した」等）。ある時点で起きた具体的な出来事として記述する。状態記述（「〜していた」「〜であった」）は不可。 |
| `normalizedLabel` | labelの要約。 |
| `category` / `subCategory` | taxonomy定義に準拠。 |

> **文体の根拠:** Fは「分岐を引き起こした具体的事象」であり、Bのような持続的状態ではない。「〜した」という完了形がこの性質を正確に表現し、Bとの区別を文体レベルで保証する。

### 2.4 個数

1イベントあたり **必須1個**。

### 2.5 カテゴリ一覧（6種）

| コード | 名称 | サブカテゴリ | 型 |
|---|---|---|---|
| F-CNF | 利害対立 | 勢力間対立、武力衝突化 | 意図的行動型 |
| F-CTD | 制度的矛盾 | 制度機能不全、財政・治安問題 | 意図的行動型 |
| F-ACT | 具体的行動 | 人物の決断・行動、死去・崩御 | 意図的行動型 / 偶発事象型 |
| F-EXT | 外圧 | 対外圧力 | 意図的行動型 / 偶発事象型 |
| F-VAC | 権力空位 | 王位空位・権力空白 | 偶発事象型 |
| F-UNC | 揺らぎ（危機・不確実性） | 災害発生、経済不安 | 偶発事象型 |

> F-ACTの「死去・崩御」(F-ACT-02) やF-EXTの一部は偶発事象型にも該当する。型の分類はカテゴリで一意に決まるのではなく、個別の事象内容で判断する。

### 2.6 代表例

**意図的行動型:**
- EV_p0592_01 F-VAC-01:「馬子が天皇の「猪の首」発言から自身への殺意を察知した」

**偶発事象型（F-UNC-01 災害発生）:**
- EV_p1732_01:「西日本でウンカの大発生により稲作が壊滅的被害を受けた」（享保の大飢饉）
- EV_p1782_01:「浅間山の大噴火により日照が減少し冷害が深刻化した」（天明の大飢饉）
- EV_p1923_01:「1923年9月1日正午、相模湾北部を震源とするM7.9の大地震が発生した」（関東大震災）
- EV_p1995_01:「1995年1月17日未明、淡路島北部を震源とするM7.3の直下型地震が発生した」（阪神・淡路大震災）
- EV_p2011_01:「2011年3月11日、三陸沖を震源とするM9.0の巨大地震と大津波が発生した」（東日本大震災）

## 3. E（Emancipation / 解放）

### 3.1 定義

B（背景の構造的条件）とF（揺らぎ）の組み合わせから生じた帰結。蓄積されたエネルギーがFをきっかけに解放される方向性を表す。

### 3.2 B×F→Eの決定性

同じFでもBが異なるとEの性質が変わる。これがBFEフレームワークの核心的性質である。

**具体例: 崇峻暗殺（592）→ 推古即位（593）**

| | 崇峻暗殺 EV_p0592_01 | 推古即位 EV_p0593_01 |
|---|---|---|
| **共通B** | 馬子が実権を掌握 (B-PWR-04) | 馬子が実権を掌握 (B-PWR-04) |
| **固有B** | 天皇との利害対立 (B-RIV-01)、天皇の独自行動 (B-PWR-02) | 継承候補の限定 (B-LEG-02)、推古の血統的適格性 (B-LEG-01) |
| **F** | 馬子が天皇の殺意を察知 (F-VAC-01) | 崇峻暗殺後の政治的空白 (F-VAC-01) |
| **E** | **先制的な権力者排除** (E-POW-03) | **協調型王権の成立** (E-POW-01) |

> 同カテゴリ (F-VAC-01) の揺らぎに対して、Bの構成が「対立型」か「協調可能型」かによって、Eが排除 (E-POW-03) と擁立 (E-POW-01) に分岐する。

### 3.3 視点ルール・記述ルール

- **実行者視点**で記述する。

| フィールド | ルール |
|---|---|
| `label` | **抽象名詞句（体言止め）** で書く（「先制的な権力者排除」「協調型王権の成立」等）。時代・人物を含まない普遍的パターン名。 |
| `normalizedLabel` | イベントの具体的表現（「崇峻天皇が蘇我馬子の意向で暗殺された」等）。 |
| `category` / `subCategory` | taxonomy定義に準拠。 |

> **文体の根拠:** Eのlabelは「何が起きたか」ではなく「どの型の構造変化か」を表すパターン名である。体言止めにすることで、(1) 異なる時代の同型イベントを比較可能にし、(2) 具体事象を記述するF（事象文）やB（状態文）と文体レベルで区別される。具体的な「誰が何をした」はnormalizedLabelが担う。

### 3.4 momentumの記述ルール

- momentum は E の解放ベクトル（方向性）を表す。
- 形式: 「〜の方向へ」（15〜40文字）。
- `data/momentum_v3_3.json` に格納。

### 3.5 個数

1イベントあたり **必須1個**。

### 3.6 カテゴリ一覧（8種）

| コード | 名称 | サブカテゴリ |
|---|---|---|
| E-POW | 権力再編型 | 皇位継承・即位、権力集中化、権力者排除・失脚、実権掌握・就任、権力空白・動揺 |
| E-REG | 体制転換型 | 王朝交代、政体転換、統治原理転換、支配構造再定義 |
| E-SYS | 制度成立型 | 新制度創設、法体系確立、官制確立、統治体系整備 |
| E-WAR | 武力衝突型 | 内戦・全国規模戦役、地方反乱・武力蜂起、決戦・会戦、対外軍事衝突、宗教・思想弾圧、宮廷政変・クーデター |
| E-ECO | 経済構造転換型 | 経済集中化、市場拡大、生産構造転換、財政基盤再編 |
| E-CUL | 文化秩序変容型 | 宗教秩序変化、価値体系転換、文化的統合、文化的分断 |
| E-ADP | 外圧適応型 | 外圧受容、制度的適応、外交均衡化、技術受容 |
| E-COL | 崩壊型 | 政体崩壊、制度瓦解、権力空白化、社会秩序崩壊 |

### 3.7 B/F/E文体の対比まとめ

| 層 | 文体 | 文末パターン | 理由 |
|---|---|---|---|
| B | 状態文 | 「〜している」「〜にある」 | イベント発生時に既に存在していた持続的条件。**現在形を正とし、過去形（「〜していた」→「〜している」）は不可。** |
| F | 事象文 | 「〜した」「〜が発生した」 | ある時点で起きた具体的な出来事（分岐点） |
| E | 抽象名詞句（体言止め） | 「〜の成立」「〜の排除」 | 時代を超えて比較可能な構造変化のパターン名 |

## 4. 設計原則

1. **Bはエネルギーを持つ** — 構造的勾配（矛盾・圧力・対立）がBの基本。土台型は例外としてOR条件で追加。
2. **Fは選択圧** — 原因でも結果でも背景でもなく、蓄積されたエネルギーの解放を方向づける揺らぎ。
3. **実行ユニットはparentFrameworkで親を参照** — 複数の実行ステップがある場合、子フレームワークが親イベントを参照する。
4. **帰属的進化はoriginで追跡** — あるイベントのEが後続イベントのBになる再帰構造（E→B連鎖）をoriginフィールドで追跡する。
5. **逆算的記述の禁止** — 結果から逆算してB/F/Eを構成しない。当時の視点で記述する。

### 5.1 B要素の土台→エネルギー型リンク

複数の土台型Bと複数のエネルギー型Bがある場合、どの土台がどのエネルギーに影響するかの関係を内部データとして保持したい。

**案1: B要素にbType + groundForフィールドを追加（自己完結）**
- B要素に `bType: "energy" | "ground"` フィールドを追加。
- 土台型Bに `groundFor: ["EL_xxx_yyy"]` でエネルギー型BのelementIdを参照。
- メリット: フレームワーク内で自己完結。
- デメリット: B要素のスキーマが複雑化。

**案2: accumulationのrelationsにgroundingメカニズムを追加（分析データとして分離）**
- `accumulation_v3_1.json` のrelationsに `mechanism: "grounding"` タイプを追加。
- メリット: 分析データとして分離でき、B要素のスキーマは変更不要。
- デメリット: 1イベント内の関係を別ファイルで管理するため参照が複雑。

**決定状況:** 未決定。どちらを採用するかは今後の仕様議論で決める。

### 5.2 縮約（折りたたみ）

### 5.3 要素構造の再検討（v3.9 議論）

> 以下は2026-03-02〜03の議論で出た課題群。v3.9仕様策定の入力とする。
> 2026-03-03: プロセスモデル（5.6）の合意により複数課題の状態が更新された。

#### 5.3.1 B/F分離基準 — **合意済み → 再考の余地あり**
- Bは状態空間・条件・制約・ポテンシャル地形（面的・持続的）
- Fは状態を動かす力（時点的・事象的）
- 両者は明確に分離する（v3.8合意）
- ただしプロセスモデル導入により、Fも「高速なプロセス」と見えるケースがある（5.6.6参照）。現時点では立ち止まらず実装段階で検証する

#### 5.3.2 F/E分離基準・Eの定義 — **方向あり、未確定**
- Fは原因側（力）、Eは帰結側（状態更新）として分離可能
- ただしEを「結果」と呼ばず、かつ固定終点にしない方向
- Eの時間範囲について議論あり（→ 5.3.7）

#### 5.3.3 Fは必ず存在するか — **合意済み**
- すべてのイベントにFは存在する（v3.8の必須1個ルールと整合）

#### 5.3.4 新構造(B-F-E)での実例提示 — **未実施**
- 定義改訂後に代表イベント数件で検証する必要あり

#### 5.3.5 taxonomy見直し — **未実施（構造確定後に仮決め）**
- 構造変更にともないカテゴリ体系の改訂が必要
- 仮決めでよいが、後の検討の基盤として必要

#### 5.3.6 Eの範囲 — **定義確定**

> **E（解放）とは、背景要素の作用状態を変化させる発火点である。**

- Eの作用は eGenerates で記述される（generate/reinforce/erode/terminate/redirect）
- 崩壊＝背景要素の作用状態をterminatedに変える、制度化＝新たな背景要素をgenerateする、意味形成＝文化型の背景要素をgenerateする
- すべて統一的に「背景要素の作用状態を変える」として扱える

#### 5.3.7 Eの時間範囲 — **定義確定**

> **Eは瞬間（発火点）であり、その後の持続は背景要素の作用状態(phases)が担う。**

- E＝発火点。背景要素の作用状態が切り替わる瞬間
- 持続＝背景要素の作用状態(phases)。emerging→established→strained→...の遷移
- 因果パターン(5.9)によりEの解釈は変わる（解放/行為帰結/適応/構造変化/新体制/連鎖帰結）が、「背景要素の作用状態を変える発火点」という本質は共通

- 以前のtaxonomyを理解した上での削減かの検証が必要

#### 5.4.6 構造変更をともなわないイベントの扱い — **定義確定**

> **すべてのイベントは何らかの背景要素の作用状態を変える。したがって、すべてのイベントがEを持てる。**

- Eは「構造変化」ではなく「背景要素の作用状態の変化」
- 構造変更がなくても、何らかの背景要素の作用状態は変わる（reinforceですら作用状態の確認・延長である）
- 例: 東大寺大仏の開眼供養 → 仏教(古代)の作用状態をreinforce。目に見える構造変化はないが、背景要素への作用は存在する

### 5.5 力学モデル（v3.9 議論）

#### 5.5.1 揺らぎの型分類 — **未決着**
- Force型（瞬間的パルス）
- Drift型（持続的ドリフト＝経時劣化含む）
- Amplify型（自己増幅的フィードバック）
- Dissipate型（自然減衰的散逸）
- 現行F-taxonomyの6種との関係を整理する必要
- F/B境界の再考（5.3.1）と連動

#### 5.5.2 吸収能力 A(t) — **新しい解釈可能、未決着**
- 相転移条件を `Tension / AbsorptionCapacity > Threshold` として定式化する案
- プロセスモデルではA(t)自体をプロセスとして扱える可能性がある
- 独立変数にするか、プロセスの属性にするか

#### 5.5.3 経時劣化（Persistent Drift） — **プロセスモデルにより解決方向**
- プロセスのライフサイクル（減衰フェーズ）そのもの
- 別メカニズムとして導入する必要はない
- プロセスが持続する中で自然に劣化・変質していく過程として統一的に扱える

### 5.6 グローバルレーン（v3.9 議論）

#### 5.6.1 基本モデル — **方向合意済み**

すべてをプロセスとして統一的に扱う。

- **原則:** すべてのBは、あるプロセス（グローバルレーン）の断面観測である
- プロセスとは「何かが動き続けている/存在し続けている状態」そのもの
- 制度・感情・思想・物理的存在物すべてを同じ構造で扱う
  - 制度が続いている＝プロセス、廃止＝プロセス停止
  - 城が存在している＝プロセス、破壊＝プロセス停止
  - 恐怖が持続している＝プロセス、忘却＝プロセス停止
- 定常状態は見かけであり、すべてにライフサイクルがある。永続的構造は存在せず、タイムスケールの違いだけ

#### 5.6.2 プロセスのライフサイクル

```
E(n)（あるFWの解放）→ プロセス駆動開始 → 持続 → B(m)として他FWに作用 → 減衰/上書き → 消滅
```

- レーンの生成: あるEがプロセスを駆動した瞬間
- レーンの持続: プロセスが動き続けている期間
- レーンの作用: 各FWでB要素として断面観測される
- レーンの消滅: プロセスが減衰、または別のEによって上書きされた

#### 5.6.3 レーン構成（v3.9）

| レーン名 | 用途 | 備考 |
|---|---|---|
| イベントレーン | タイムライン上のイベント表示 | 現行通り |
| 因果レーン | 各FW内のE→F→B要素を配置（1本） | 表示順を逆転（v3.8まではB→F→E） |
| 背景レーン | プロセスのライフサイクル表示 | **新設** |

因果レーンの表示順（v3.9）:
```
[イベントレーン]        ← イベント（タイムライン）
[因果レーン: E → F → B] ← Eがイベント側、Bが背景側
[背景レーン: プロセス]   ← Bの本体（プロセスのライフサイクル）
```
Eをイベントレーン側に配置することで「イベント＝解放の観測点」の関係が直感的になる。Bを背景レーン側に配置することで「B＝プロセスの断面観測」の関係が視覚的に接続される。

モデルとUIの分離:

| レベル | 扱い |
|---|---|
| モデル（データ構造） | すべてのBにプロセス（レーン）への帰属がある |
| 表示（UI） | 複数イベントにまたがって観測されるプロセスだけを背景レーンとして描画する |

背景レーンの表示条件: 同じプロセスが異なるイベントのB要素として2回以上出現する場合。解像度が粗い場合に1度しか登場しないものは背景レーンに表示しない。

#### 5.6.4 構造の向き

レーン（プロセス）が本体であり、B要素はその断面観測（ポインタ）。

```
従来の発想:  B要素（本体） ← レーンが参照する
正しい構造: プロセス/レーン（本体） ← B要素はその断面観測
```

#### 5.6.5 Bの本質の再定義

Bは「静的な状態」ではなく「動き続けているプロセスそのもの」が背景として作用している。

- Bはプロセスの「結果」ではない。プロセスが動いている状態＝B
- すべてのBはどこかのEに駆動されたプロセスに由来する
- 現行のoriginフィールドはこの構造の部分的な先行実装だが、ライフサイクル全体は未表現

#### 5.6.6 F/B境界の再考

プロセスモデルを前提にすると、FとBの区別が曖昧になるケースがある。

- 逓増的に加速度的に圧が高まる現象 → 構造的にはBの蓄積と区別がつかない
- 短時間で高まるエネルギー → 揺らぎのように見えるが、高速なプロセスとも言える
- v3.8で合意した「B=状態、F=力」の分離は有効だが、時間スケールの違いでしかないケースがある
- 現時点ではこの問題で立ち止まらず、実装段階で具体例から検証する

#### 5.6.7 背景レーンの表示制約

- 同時表示は最大5本まで。モデル上はプロセス数に制限なし。

#### 5.6.8 リンクタイプ分類

##### A. イベント → プロセスへの作用

| タイプ | 意味 | 例 |
|---|---|---|
| generate | プロセスを新たに駆動する | 大宝律令 → 律令的土地制度を生成 |
| reinforce | プロセスを強化・加速する | 班田収授の実施 → 律令制を強化 |
| erode | プロセスを弱める・変質させる | 墾田永年私財法 → 律令制を侵食 |
| terminate | プロセスを停止させる | 太閤検地 → 荘園制を終了 |
| redirect | プロセスの方向を変える | 延久荘園整理令 → 荘園制に抑制方向の力 |

##### B. プロセス → イベントへの作用（B断面観測の性質）

| タイプ | 意味 | 例 |

##### C. プロセス内部の動態

| タイプ | 意味 | 例 |
|---|---|---|
| gradual | 漸進的変化（だんだんに） | 荘園がじわじわ拡大 |
| sudden | 急激な変化（一気に） | 大宝律令で制度が一気に成立 |

| decay | 自然な減衰 | 制度が運用されなくなり形骸化 |

##### D. プロセス間の相互作用

| タイプ | 意味 | 例 |
|---|---|---|
| inverse | 逆相関（一方の成長が他方の衰退） | 荘園制↑ ↔ 律令的土地制度↓ |
| driving | 一方が他方を駆動 | 律令制の疲弊が荘園制を駆動 |
| competing | 同じ領域で競合 | 公地制 vs 私有制 |

### 5.8 背景と背景要素の分離モデル（v3.9 確定方向）

> 2026-03-07 確定。デモ `doc/demo_background_lane_v3.html` およびサンプルデータ `data/v3_9_sample_5events.json` で検証済み。

#### 5.8.1 三つの概念

v3.8までの「B（背景）」を3つの概念に分離する。

| 概念 | 位置 | 性質 | 例 |
|------|------|------|-----|
| **背景** | 因果FW内（B層） | 背景要素が背景ミニFWに組み込まれて生じる圧・歪み・矛盾 | 「口分田不足による財政・民生の圧」 |
| **背景要素** | 背景レーン | 構造的因子。単独では作用の方向を持たない | 「律令制」「班田収授制」「荘園制」 |
| **背景ミニフレームワーク** | 因果FW内（B層の構造） | 背景要素を組み込んで背景を生成するしくみ | 「制度的矛盾→圧」型、「外圧→対応圧」型 |

**背景要素は素材であり、それ自体は方向を持たない。** 背景ミニフレームワークが背景要素を受け入れるインターフェースを持ち、組み込まれて初めて具体的な背景（圧・歪み）として因果レーンに現れる。

#### 5.8.2 循環関係

```
背景要素 ──[インターフェース]──▶ 背景ミニFW ──▶ 背景(B) ──F──▶ E(解放)
   ▲                                                           │
   └──────────────── eGenerates ◀──────────────────────────────┘
```

1. 背景ミニFWがインターフェースを通じて背景要素を受け入れ、具体的な背景（圧・歪み）を生む
2. Fが背景を解放してEが発生する
3. Eが既存の背景要素を変化させる（生成/強化/侵食/終了/転換）、または新たな背景要素を生む
4. 変化した背景要素が次の背景ミニFWに組み込まれ、次の背景を生む → 繰り返し

#### 5.8.3 背景の定義（v3.8 セクション1を改訂）

背景は**背景要素が背景ミニフレームワークに組み込まれて生じるエネルギー**である。

- 「圧」「歪み」「矛盾」「不満」「危機感」「期待」などの力学的状態
- 背景ミニFWのインターフェースがどの背景要素を組み込んでいるかを明示する
- 1イベントあたり1〜3個（v3.8の2〜5個より減る。土台型Bが背景要素に移行するため）
- v3.8のB要素のうち「エネルギー型」が背景、「土台型」が背景要素に対応する

記述ルール:
- `label`: 「〜の圧」「〜への危機感」「〜の歪み」等。力学的表現を使う
- `contributingBG`: 背景要素のIDリスト（背景ミニFWのインターフェースが受け入れた背景要素）
- `note`: 背景ミニFWがどの背景要素をどのように組み合わせてこの背景を生んでいるかの説明

#### 5.8.4 背景要素の定義

背景要素はイベントの背後にある**構造的因子**である。それ自体は作用の方向を持たないが、背景ミニFWに組み込まれて初めて具体的な背景として作用する。

##### タイプ分類

| type | 名称 | 例 |
|------|------|-----|
| institution | 制度 | 律令制、班田収授制、荘園制、国司行政 |
| knowledge | 知識 | 唐制を範とすべき知識(normative)、土地政策の成り行き知識(strategic) |
| demographic | 人口・地理 | 人口増加圧、耕地面積制約 |
| power_structure | 権力構造 | 藤原氏摂関体制、院政体制 |
| culture | 文化・価値 | 仏教思想、武士の倫理 |
| external | 対外環境 | 唐との関係、蒙古の脅威 |
| resource | 資源基盤 | 財政基盤、軍事力 |
| conflict | 利害対立構造 | 公家と武家の対立、守護と国人の対立 |

##### 背景要素の時系列情報

各背景要素は2つの時系列情報を持つ:

**作用イベントログ (effectLog)** — この背景要素に作用したイベントの事実記録。eGeneratesの逆引きであり、自動生成される。
```json
"effectLog": [
  {"eventId": "EV_p0701_01", "year": 701, "effect": "generate"},
  {"eventId": "EV_p0743_01", "year": 743, "effect": "erode"}
]
```

**作用状態 (phases)** — 作用イベントログから読み取れる背景要素の状態変化の解釈。期間ごとに背景要素がどういう位相にあったかを記述する。
```json
"phases": [
  {"period": "701-743", "phase": "established", "label": "成立・本格運用"},
  {"period": "743-1069", "phase": "hollowing", "label": "形骸化進行"}
]
```

phase語彙（暫定）: `emerging` / `established` / `expanding` / `strained` / `hollowing` / `declining` / `nominal` / `terminated`

作用イベントログは事実、作用状態は解釈。作用イベントログから自動的に作用状態を推論することは可能だが、人間による検証が望ましい。

##### 背景要素の同一性

異なるイベントで参照される背景要素が「同じもの」かどうかは `bgId` で判定する。

#### 5.8.5 E→背景要素リンク（eGenerates）

Eが発生した結果、背景要素に及ぼす影響を記述する。

| effect | 意味 | 例 |
|--------|------|-----|
| generate | 新たな背景要素を生成 | 大宝律令(E) → 律令制を生成 |
| reinforce | 既存の背景要素を強化 | 大宝律令(E) → 班田収授制を強化 |
| erode | 既存の背景要素を侵食・弱化 | 墾田永年私財法(E) → 律令制を侵食 |
| terminate | 既存の背景要素を終了 | 墾田永年私財法(E) → 班田制を終了 |
| redirect | 既存の背景要素の方向を変更 | 延久荘園整理令(E) → 荘園制に整理圧 |

1イベントあたり1〜4個。

#### 5.8.6 背景要素→背景リンク（背景ミニFWのインターフェース）

背景ミニFWがインターフェースを通じて背景要素を受け入れ、背景（圧・歪み）を生成する。
データ上は背景の `contributingBG` フィールドが背景要素のIDリストを保持する。
各背景要素が背景にどのように寄与するかは `note` フィールドで説明する。

#### 5.8.7 v3.8 B要素からの変換ルール

v3.8の各イベントのB要素（elements where layer="B"）を以下のように振り分ける:

| v3.8のB要素の性質 | v3.9での行き先 | 判定基準 |
|-------------------|---------------|----------|
| 圧・歪み・矛盾・対立 | 背景 | label が「〜している」で力学的緊張を含む |
| 制度・構造・存在の記述 | 背景要素 | label が存在事実・構造記述で、エネルギーを含まない |
| 混合（構造+圧） | 分解 | 構造部分→背景要素、エネルギー部分→背景 |

典型的パターン:
- v3.8: B要素3個（土台1+土台1+エネルギー1）→ v3.9: 背景要素2個 + 背景1個(contributingBG=[BG1,BG2])
- v3.8: B要素3個（全部エネルギー型）→ v3.9: 複数の背景要素を抽出 + 背景1〜2個

### 5.10 コンポーザブル因果フレーム（v3.9 確定方向）

> 2026-03-08 設計・実装。パターンタグ制（5.9）を進化させ、固定B-F-Eスロットを関数型フレームに置換する。

#### 5.10.1 設計思想

固定の3スロット（B-F-E）にパターンタグを付ける方式では、以下の問題が残った:
- **F/E分離基準**: パターンごとにFの意味が異なるなか統一基準が定まらない
- **F必須性**: 権力移行型で「後継手続」が存在しないスムーズな移行がある
- **3層 vs 2層**: 背景ミニFWの導入で背景の内部構造が豊かになり、固定3層が制約に
- **A(t)**: 吸収能力が独立変数なのか背景の文脈依存属性なのか不明

コンポーザブル因果フレームはこれらを一挙に解決する:

```
<Fname>(param1: value1, param2: value2, ...): result
```

- **Fname** が因果構造を決定する（固定スロットではなくフレームが構造を定義）
- **パラメータのロール・数・必須性** はFnameごとに定義される
- **第1パラメータは常にBname型** — 背景合成フレームをネストして背景要素→背景の変換を行う
- **resultは常に「解放の一般化」** — 背景要素の作用状態を変化させる発火点

#### 5.10.2 因果フレーム（Fname）— 6種

イベントの因果構造を定義するトップレベルの関数。

| Fname | シグネチャ | 件数 |
|-------|----------|------|
| 圧力解放 | `圧力解放(背景: Bname, 揺らぎ: text): 解放` | 63 |
| 主体的行為 | `主体的行為(状況: Bname, 判断: text): 帰結` | 58 |
| 矛盾露呈 | `矛盾露呈(潜在矛盾: Bname, 顕在化契機: text): 構造変化` | 48 |
| 外生衝撃 | `外生衝撃(受容状態: Bname, 衝撃: text): 適応` | 21 |
| 偶発連鎖 | `偶発連鎖(脆弱性: Bname, 偶発事象: text): 連鎖帰結` | 9 |
| 権力移行 | `権力移行(権力空白: Bname, 後継手続?: text): 新体制` | 8 |

共通構造:
- 第1パラメータ: **Bname型**（背景合成フレーム）— 背景要素を背景に変換
- 第2パラメータ: **text型**（旧F層に相当）— 省略可能なものは`?`マーク
- result: **解放の一般化** — 背景要素のphasesを変化させる発火点（旧E層に相当）

#### 5.10.3 背景合成フレーム（Bname）— 8種

背景要素を組み合わせて具体的な「背景」を生成する関数。Fnameの第1パラメータにネストされる。

| Bname | シグネチャ | 件数 |
|-------|----------|------|
| 権力集中 | `権力集中(権力体: BG): 集権的支配状態` | 43 |
| 外圧環境 | `外圧環境(国際環境: BG, 国内状態?: BG): 外圧` | 42 |
| 勢力対立 | `勢力対立(勢力: BG[]): 対立の圧` | 27 |
| 思想潮流 | `思想潮流(思想: BG, 社会状態?: BG): 思想的圧力` | 22 |
| 経済圧 | `経済圧(経済構造: BG, 階層?: BG): 経済的緊張` | 21 |
| 複合背景 | `複合背景(要素: BG\|Bname[]): 複合的背景` | 19 |
| 制度疲弊 | `制度疲弊(制度: BG): 制度的空洞化` | 18 |
| 制度的矛盾 | `制度的矛盾(要素: BG[]): 制度間の緊張` | 15 |

BG = backgroundElementsのbgIdへの参照。`?`付きは省略可能。`[]`は複数可。

#### 5.10.4 ネスティングと合成規則

```
因果フレーム全体:
  Fname(
    Bname(BG_ref, BG_ref, ...),   ← 背景合成フレーム（ネスト）
    "テキスト記述"                  ← 旧F層
  ): 解放テキスト                   ← 旧E層
```

- **Fnameの背景パラメータにBnameをネスト可能**
- **Bnameのパラメータに別のBnameをネスト可能**（深さ制限なし）
- **複合背景**: 他のBnameの出力を組み合わせる汎用パターン
- **resultの意味**: Fnameの定義に従う（解放/帰結/適応/構造変化/新体制/連鎖帰結）

#### 5.10.7 背景要素分離モデル(5.8)との関係

コンポーザブル因果フレームは背景要素分離モデルと完全に整合する:

- **背景レーン**: フレーム種別に関係なく共通（背景要素の作用状態遷移）
- **eGenerates**: フレーム種別に関係なくresult→背景要素への影響は普遍的
- **Bname**: 背景ミニFWの「しくみ」の具体的な型付け。背景要素→背景の変換ロジックを明示する
- **Fname**: イベント全体の因果構造。Bnameの出力とtextパラメータからresultを導く

#### 5.10.8 解決した課題

| ref | 課題 | 解決方法 |
|---|------|---------|
| 5.3.2 | F/E分離基準 | Fnameがパラメータロールを定義。「何がFで何がEか」はフレーム定義から自動決定 |
| 5.3.3 | Fは必須か | Fname定義で`?`マーク。権力移行の後継手続は省略可能 |
| 5.4.1 | 3層 vs 2層 | 固定層の概念自体が不要。Fnameがパラメータ数と構造を決定 |
| 5.5.2 | 吸収能力A(t) | Bnameのパラメータとlabelが文脈依存の背景を生成。A(t)は背景合成の副産物 |

#### 5.10.11 v4.0 設計方針（将来構想 → 一部実装済み）

v3.9のコンポーザブル因果フレームは固定2パラメータ構造（Bname型 + text型）を採用していたが、
functional_v2で以下を実装済み。残りは将来構想として維持する。

**1. 可変長型付きパラメータ** — ✅ 実装済み

paramsを `{value, type}` の配列とし、個数を可変にした。Fnameごとに異なるパラメータ数・型を許容。

```json
"params": [
  {"value": "権力が豪族勢力に集中している", "type": "権力集中状態"},
  {"value": "武烈天皇が後嗣なく崩御し大王位が空位となった", "type": "権力者不在による権力空白"},
  {"value": "豪族間の合議を経て遠縁の継体が擁立された", "type": "正統性ある後継者の擁立"}
]
```

**2. 関数本体（因果メカニズム記述）** — ✅ 実装済み

`body`フィールドに`[%1]...[%N]`テンプレートでparamsを参照する因果メカニズムを記述。

```json
"body": "[%1]のもと[%2]が生じ => [%3]"
```

**3. fname抽象化** — ✅ 実装済み

固有名詞を含まない汎用因果パターン名に抽象化済み。`fnameCategory`（6種）と具体的`fname`の2層構造。

**4. background復元** — ✅ 実装済み

Bname型の背景合成を `background` フィールドとして分離。`{bname, refs[], label}` 形式で背景要素への参照を保持。

**5. 型付き返り値**

返り値にも型を与える。v3.9では文字列ラベルだが、v4.0では構造体または関数型を検討する。

- **構造体型:** `制度転換("墾田永年私財法")` — ラベル付き型
- **関数型:** 返り値自体がBnameとなり、次のイベントのパラメータに直接接続可能

**4. Lisp的関数合成によるイベント連鎖**

返り値を次のイベントの引数とすることで、イベント間の因果連鎖を関数合成として表現する。

```lisp
;; 概念例: 墾田永年私財法 → 荘園制確立 → ...
(矛盾露呈
  (圧力解放
    (制度疲弊 律令土地制度の形骸化)
    (経済圧 荘園拡大))
  (権力集中 藤原氏の荘園集積))
```

これにより、個別イベントの因果構造と、イベント間の因果連鎖を同一の形式言語で記述できる。

**5. eGeneratesの位置づけ**

generate/reinforce/erode/terminate/redirect は関数の**副作用（side effects）**として扱う。
返り値（主たる帰結）とは区別される。

```
圧力解放(...) { ... }: 制度転換("墾田永年私財法")
  ├─ generate: BG_shoen_system       // 副作用
  ├─ erode: BG_ritsuryou_land        // 副作用
  └─ reinforce: BG_kokushi_power     // 副作用
```
