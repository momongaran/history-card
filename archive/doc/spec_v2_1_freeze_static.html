myHistory 仕様 v2.1 Freeze（静的UI / 揺らぎF必須1）

実装確定版 — 2026-02-21

1. このFreezeの狙い

  • 「確定した部分だけ」をFreezeし、作り手と表示側の共通認識を揃える。
  • 未確定（今回対象外）：グローバルレーン、6軸円環、動的演出（ホバー線/アニメ等）、対応リンクの描画ルール。

2. データモデル（FrameworkView）

2.1 1イベント = 1 FrameworkView

  • 各イベント eventId に対し、frameworkViews は必ず1つ存在する。

2.2 レイヤー（layer）

既存のレイヤー体系（C/P/R）を維持しつつ、揺らぎ専用レイヤー F を追加する。

layer意味（Freeze時の扱い）
CContext（背景）。既存taxonomyのContext群（C-*）をそのまま継承する。
FFluctuation（揺らぎ）。必須1（Primary Fluctuation）。
PPath（経路）。データとして存在してよいが、Freeze静的UIの必須対象にはしない（表示してもしなくてもよい）。
RResult（結果）。既存taxonomyのResult群（R-*）をそのまま継承する。

2.3 必須制約（Freeze）

レイヤー必須数備考
C（背景）1以上既存資産を維持
F（揺らぎ）ちょうど1Primary Fluctuation。複数Fは将来拡張（未決定）
R（結果）1以上既存資産を維持

2.4 Fが欠けるFW

  • Fが欠けるFWはエラー扱い。データ側で補完済み。未設定は F-UNC-00。

3. taxonomy（カテゴリ体系）

  • taxonomy_v2_1_freeze.json は、既存の C/P/R/L を切り落とさず維持した上で、F を追加する。
  • 揺らぎは旧 C-UNC から切り出し、F-UNC として移動する（サブカテゴリも移植）。
  • 画面表示・ラベル参照は新taxonomy（taxonomy_v2_1_freeze.json）を正とする。

4. 理論注記：吸収能力 A(t)

吸収能力 A(t) は「背景（C）の状態変数」として仮定しうる。

ただし Phase1 Freeze では A(t) を独立要素としてデータ化・可視化しない（注意書きとして保持）。

将来、背景に変化を与えるフレームワーク（背景変化FW）として表現する可能性がある。

5. 対応リンク（correspondences）

  • リンクカテゴリ（L）は既存taxonomyを維持する。
  • default_correspondences.json は現状のまま利用可。
  • Freeze静的UIでは「イベント↔要素を結ぶ線」の描画を行わない（表示しない）。
  • 理由：特に長期揺らぎ等は、直線的リンクだけで意味を確定させるのに無理が出やすいため。
  • 対応リンクデータは保存のみ（保存/読込機能は維持）。

6. UI仕様（静的Freeze）

6.1 基本方針

  • 装飾を削り、情報階層を視覚強度で表す。
  • 「紙面」のような静けさを優先する。
  • 白背景固定。グラデーション・影（box-shadow）・立体感は禁止。

6.2 レイアウト

  • 左カラム：イベント行（年号 / タイトル）
  • 右カラム：当該イベントの因果要素コードを横並びで表示
  • 左右カラム境界：1px solid #eee の薄線のみ

6.3 因果要素の表示

項目仕様
表示形式テキストspan。バッジ（背景色・枠線・角丸・点線）は使用しない。
表示例PWR02 LEG01 UNC01 SUC01 POW01
表示順序C → F → P（任意） → R
フォント等幅フォント（SF Mono / Menlo / Consolas）、11px、font-weight: 500
垂直位置イベント行の中央揃え（align-items: center）。baseline揃え禁止。
要素間隔gap 8px 固定。可変余白は使わない。
レイヤー境界C/F/P/R の境界に margin-left: 12px の余白を追加。区切り線は使わない。

6.4 文字色による視覚階層

彩度は使わない。識別目的の最小限の濃度差のみ。

要素文字色意図
F（揺らぎ）#222やや濃い — 揺らぎを前景に
C（背景）/ P（経路）/ R（結果）#444標準
F-UNC-00（揺らぎ未設定）#bbb薄灰 — 視覚階層を一段下げる

6.5 F-UNC-00（揺らぎ未設定）の扱い

  • 表示は維持する（省略しない）。
  • サイズ・形状は他の要素と同一。
  • 点線・背景色は使わない。
  • 薄灰色テキスト（#bbb）のみで視覚階層を下げる。

6.6 動的演出

  • ホバー時：控えめなアンダーライン表示（text-decoration: underline; color: #ccc）。
  • ホバー時ツールチップ：要素の詳細情報（コード・サブカテゴリ名・ラベル）を表示。
  • SVGリンク線の描画：なし。
  • アニメーション：なし。

6.7 禁止事項

  • 背景色面によるバッジ表現
  • 枠線・角丸・点線
  • グラデーション・box-shadow・立体感
  • ダークモード
