myHistory UI仕様 v2.0

更新日: 2026-02-19 / 前版: v1.1 Freeze (2026-02-14)
本版は R05 実装（横並びバッジ版）に合わせた仕様更新。変更箇所は 黄色背景、新規は 緑色背景 で示す。

1. 目的

本仕様書は、歴史イベントを並べたタイムライン上で「Framework View（FW）」の因果要素を表示・編集するための、データ構造およびUI表示の共通ルール（正本）を定義する。

2. 用語

  • Event: 歴史イベント。イベントレーンに縦方向に並ぶ。
  • FrameworkView（FW）: 1イベントに対応する因果の枠組み。1イベント = 1FW。
  • Element: FWを構成する要素。layer（C/P/R）と分類コード（category/subCategory）を持つ。
  • taxonomy.json: 分類コード→表示名の辞書。カテゴリ/サブカテゴリ表示名称の唯一の正本。
  • バッジ（Badge）: 要素をコンパクトに表示する横長チップ。レイヤー色で色分けされる。
  • 対応リンク（Correspondence）: ある要素を別のイベントに紐付ける関係。Lコードで性質を分類する。

3. スコープ（Phase 1.0）

3.1 含む

  • taxonomy.json に基づくカテゴリ/サブカテゴリ表示
  • FrameworkView（1イベント = 1FW）の表示
  • FW要素の横並びバッジ表示（イベント行ごとにフラット展開）
  • 対応リンクの作成・編集・削除（Lコード分類）
  • Undo/Redo（Ctrl+Z / Ctrl+Shift+Z）
  • 保存・読込（JSON）
  • 統計分析パネル

3.2 含まない（後続フェーズ）

  • transition（順序・分岐）の定義/描画
  • ドラッグ＆ドロップ配置、スナップ接続
  • 複合FW、FW同士の対応リンクによるメタ構造
  • 反実仮想/最適化/シミュレーション

4. データ仕様（FrameworkView JSON v2.0 Freeze）

4.1 ルート構造

{
  "events": [ { "eventId", "title", "sortKey", "endKey", "era", "trails", "note" } ],
  "frameworkViews": [ { "frameworkViewId", "eventId", "lanes", "elements", "meta", "version" } ]
}

4.2 events（イベント）

フィールド型説明
eventIdstring一意識別子（"EV_..."）
titlestringイベント名称
sortKeynumber年代（西暦。BC は負数）
endKeynumber | null終了年（範囲がある場合）
erastring | null時代名称
trailsstring[]トレイルタグ
notestring | null注記

4.3 lanes（因果レーン）

{
  "laneId": "LN_1",
  "laneIndex": 1,
  "label": "因果レーン #1",
  "elements": ["EL_1", "EL_2", "EL_3"]
}
  • elements: このレーンに属する elementId の配列。表示順序を決定する。

4.4 elements（要素）

{
  "elementId": "EL_1",
  "layer": "C",
  "category": "C-LEG",
  "subCategory": "C-LEG-02",
  "label": "武烈天皇の死で王統が不安定化"
}
フィールド説明
layer"C"（Context） / "P"（Path） / "R"（Result）
categorytaxonomy.json のカテゴリコード（例: C-LEG）
subCategorytaxonomy.json のサブカテゴリコード（例: C-LEG-02）
labelUIに表示する短文

4.5 correspondences（対応リンク — UIステート）

対応リンクはデータファイルではなく、UIの状態として管理される。保存/読込でJSON化。

{
  "[fwViewId]": {
    "[elementId]": {
      "eventId": "EV_...",
      "category": "L-CAU",    // Lコード
      "subCategory": null
    }
  }
}

4.6 taxonomy.json（共通辞書）

  • taxonomy.json はカテゴリ/サブカテゴリ表示名称の唯一の正本
  • Element側には名称を持たせず、UIで参照して補完表示する
  • Lコードも layers.L に定義される

5. UI仕様（正本）

5.1 レイアウト（2カラム）

v1.1では3カラム（イベント / ガター / 因果レーン）だったが、v2.0ではガターを廃止し2カラムに変更。

  • 左カラム: イベントレーン（固定幅 380px、縦スクロール）
  • 右カラム: 因果要素エリア（残り幅、縦スクロール同期）
.top {
  display: grid;
  grid-template-columns: 380px 1fr;
  gap: 8px;
}

5.2 因果要素の表示方式（横並びバッジ）

v1.1ではFWを因果レーン内に縦配置していたが、v2.0ではイベント行ごとにFWの全要素を横並びバッジとしてフラット展開する。

  • 各イベント行に、そのイベントのFW要素がバッジとして横に並ぶ
  • バッジの順序は lanes[0].elements の配列順（C → P → R）
  • 連番はFW単位で1から振られる
  • 因果レーンの概念は表示上は使用しない（データ上は保持）

5.3 バッジ表示

要素はコンパクトなバッジとして表示する。v1.1の2行FWボックスに替わるもの。

  • 表示テキスト: subCategoryコードからレイヤープレフィクスとハイフンを除去（例: C-PWR-02 → PWR02）
  • バッジ色はレイヤーで決定:
    • C層: 青系（#f0f4fa / border #c7d4e8 / 文字 #3d5a8a）
    • P層: 茶系（#faf5ed / border #ddd0b5 / 文字 #7a6840）
    • R層: 緑系（#eff6f0 / border #bad4bc / 文字 #3a6b3e）
  • 未リンク要素: ゴースト表示（破線ボーダー、薄い文字色）
  • リンク済み要素: 左ボーダー 3px アクセントカラー
  • フォント: モノスペース（SF Mono / Menlo / Consolas）10.5px

5.4 ホバーツールチップ

バッジにマウスホバーすると詳細ツールチップを表示する:

  1. 連番（01〜）+ subCategoryコード + サブカテゴリ名称（taxonomy参照）
  2. label本文
  3. Lコード（リンク済みの場合）
  4. リンク先イベント名 + 距離（行差）

ホバー中、リンク先イベント行までSVGベジエ曲線を描画する。

5.5 年代表示

  • 単一年: 数値のみ（例: 593）
  • 範囲年: 開始 – 終了（例: 1603 – 1868）
  • BC: BC プレフィクス（例: BC660）
  • 年代がアバウト（概数）であることを示す場合のみドット（.）を付加する。確定年にはドットを付けない
  • 現状のデータには概数フラグがないため、ドットは一切表示しない

5.6 イベント行

項目値
行高--row-height: 64px（3行表示可能）
グリッド88px 1fr 20px（年 / タイトル / expander）
タイトル最大3行（-webkit-line-clamp: 3）
区切りborder-bottom: 1px solid #f0f1f3

5.7 行高の同期

イベントレーンと因果要素エリアの行高は動的に同期する。バッジが折り返した場合、両側の行高が自動調整される（syncGridRows）。

5.8 スクロール同期

左右パネルの縦スクロールは requestAnimationFrame で同期する。

6. 対応リンク仕様

6.1 Lコード（対応リンクカテゴリ）

対応リンクのカテゴリは、要素自身の分類コード（C/P/Rコード）ではなく、リンクの性質を表すLコードを使用する。

コード名称意味
L-CAU直接因果AがBを直接引き起こした
L-ICU間接因果AがBに間接的に影響した
L-PRQ前提条件AがBの前提となった
L-RPL波及Aの影響がBに波及した
L-TRG誘発AがBを誘発した
L-SIM類似構造AとBは構造的に類似する
L-CNT対比AとBは対比関係にある
L-TRN転用Aの仕組みがBに転用された
L-SUP上位包含AはBの上位概念である
L-SUB下位展開AはBの下位展開である

6.2 リンク操作フロー

  1. リンク作成: 未リンクバッジをクリック → イベント行をクリック → リンク成立
  2. Lコード変更: リンク済みバッジをクリック → ドロップダウンでLコード選択
  3. イベント再選択: ドロップダウン内「↻ イベント再選択」
  4. リンク削除: リンク済みバッジを右クリック → 確認ダイアログ
  5. キャンセル: Esc / 関係ない場所をクリック

6.3 Undo/Redo

  • Ctrl+Z: Undo（最大50回）
  • Ctrl+Shift+Z: Redo
  • 新しい操作でRedo履歴はクリアされる

7. 統計パネル

「統計」ボタンでモーダルオーバーレイを表示。以下の4セクション:

  1. サマリーカード: イベント数 / FW数 / 要素数 / リンク済み数
  2. レイヤー構成: C/P/R の割合バー
  3. リンクカテゴリ分布: Lコード別の件数バー
  4. 因果パターン分析: FW内要素パターンの出現頻度（2回以上のもの）

11. v1.1 → v2.0 変更履歴

項目v1.1v2.0理由
レイアウト 3カラム（イベント / ガター / 因果レーン群） 2カラム（イベント / 因果要素バッジ） 要素を横並びにすることで一覧性を向上
要素表示 因果レーン内に縦配置、2行FWボックス イベント行ごとに横並びバッジ コンパクトで直感的な表示
ガター 独立カラム（72px固定幅）にリンク情報表示 廃止（バッジの色・スタイルでリンク状態を表現） バッジがリンク状態を直接表現するためガター不要
因果レーン FWをレーン単位で縦に配置、横スクロール レーン概念を表示上は使用しない 1イベント=1FWのため、レーン分離のメリットが薄い
年代ドット （未定義） 確定年にはドット不要。概数の場合のみドット付与（現データでは一律なし） 通常の年表表記に準拠
統計パネル （なし） パターン分析・レイヤー構成・Lコード分布 分析機能の追加
ホバーツールチップ （なし） バッジホバーで詳細表示 + SVGリンク線 コンパクト表示の詳細補完
保存/読込 （なし） 対応リンクのJSON保存/読込 作業の永続化
