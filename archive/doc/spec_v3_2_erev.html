myHistory 仕様 v3.2（E要素ラベル書き換え + legacyResult臨時保管）

実装確定版 — 2026-02-23

1. 概要

v3.1 の accumulation 追加後、E要素（Emancipation）のラベルは旧R（Result）データのまま「帰結・結果」を記述していた。 v3.2 では E-label を「eventタイトルを一段抽象化した記述」（解放の瞬間）に書き換え、旧ラベル（帰結記述）を notes.legacyResult に臨時保管する。

1.1 目的

  • E要素の意味を「帰結・結果」から「解放の瞬間＝蓄積されたエネルギーが発現する瞬間」に転換する
  • 固有名詞を除去し、構造的・類型的な表現にすることで、パターン比較・継承リンクの材料にする
  • 旧ラベルは将来の継承リンク・上位FW等の材料として臨時保管する

2. E-label書き換えルール

2.1 抽象化の原則

  1. 固有名詞（人名・地名・年号）を除去し、構造的・類型的な表現にする
  2. eventタイトルの主語・動詞・対象から、E-categoryに応じた抽象的表現を生成する
  3. 「解放の瞬間」＝蓄積されたエネルギーが発現する瞬間として読める表現にする
  4. 15〜25文字程度の簡潔な記述にする

2.2 カテゴリ別抽象化パターン

E-category名称抽象化の方向変換例
E-POW権力再編型権力構造の変動を表す語「継体天皇が即位した」→「王統断絶からの新王擁立」
E-WAR武力衝突型軍事的事態を表す語「磐井の乱が起こった」→「地方有力者の武力蜂起」
E-SYS制度成立型制度的変革を表す語「冠位十二階を制定した」→「個人評価に基づく官位制度の創設」
E-REG体制転換型体制変動を表す語「白河上皇が院政を開始した」→「退位君主による統治形態の確立」
E-CUL文化秩序変容型文化的変容を表す語「百済から仏教が伝来した」→「外来宗教体系の到来」
E-ADP外圧適応型対外適応を表す語「ペリーが来航した」→「外圧による対外対応の迫り」
E-ECO経済構造転換型経済変動を表す語「和同開珎が鋳造された」→「貨幣制度の導入」
E-COL崩壊型崩壊・瓦解を表す語「関東大震災が起きた」→「大規模自然災害による社会危機」

2.3 動詞パターンマッチング

eventタイトルの末尾動詞パターンにより、E-categoryと組み合わせて抽象表現を選択する。主要パターン:

パターンE-SYS での変換例E-WAR での変換例
〜を制定した新制度の創設—
〜が起きた制度的変革の開始軍事的衝突の発生
〜が出された統治方針の制度化—
〜が成立した新制度体系の成立—
〜の乱が起きた—武力蜂起による秩序の動揺
〜の変が起きた—政変による権力構造の転換
〜が滅亡した—武力による政権の壊滅

3. legacyResult臨時保管

旧E-label（帰結記述）は notes.legacyResult に保存する。legacyPath と同じパターン。

{
  "notes": {
    "legacyPath": [
      {
        "elementId": "EL_EV_p0507_01_3",
        "category": "P-SUC",
        "subCategory": "P-SUC-01",
        "label": "継体天皇として即位し大和入りに約20年を要した"
      }
    ],
    "legacyResult": "応神天皇五世孫として王統の正統性が主張された"
  }
}

3.1 notes フィールド

フィールド型説明
notes.legacyPatharrayv3.0 で退避した旧P要素（v3.0以降存在）
notes.legacyResultstringv3.2 で退避した旧E-label（帰結記述）。将来の継承リンク・上位FW等の材料。

5. 中間データ形式

生成スクリプトは中間ファイル emancipation_revisions_v3_2.json を出力し、マージスクリプトで本体に統合する。

{
  "version": "3.2",
  "generatedBy": "revise_emancipation.py",
  "model": "rule-based",
  "timestamp": "2026-02-23T12:34:00",
  "items": {
    "FW_EV_pXXXX_XX": {
      "elementId": "EL_...",
      "originalLabel": "旧E-label（帰結記述）",
      "revisedLabel": "新E-label（抽象化記述）",
      "categoryRevised": "E-POW",
      "subCategoryRevised": "E-POW-01"
    }
  }
}

6. バリデーションルール

  • 全221件のE要素に label が存在すること
  • E-category が有効な値（8種）であること
  • notes.legacyResult に旧ラベルが保管されていること（221件全て）
  • 新E-label が eventタイトルの抽象化として妥当であること（目視サンプル確認）
  • 新E-label の長さが 5〜25文字程度であること
  • ブラウザ表示が壊れないこと
