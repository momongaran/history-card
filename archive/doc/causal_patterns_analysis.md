# 因果パターン多元化分析

> 2026-03-08 作成
> v3.8データ207件のE×Fカテゴリ分析に基づく

## 1. 問題

現行B-F-E（圧力蓄積→引き金→解放）は単一の因果テンプレートを全イベントに強制している。
v3.8データの分析により、**B-F-Eが自然にフィットするのは全207件中43件（21%）** に過ぎないことが判明。

| フィット度 | 件数 | 割合 | 説明 |
|-----------|------|------|------|
| 自然 | 43 | 21% | B(圧力)要素にエネルギー語あり + F(衝突/矛盾)で発火 |
| やや無理 | 62 | 30% | 軍事/矛盾だがB要素に圧力語がない |
| 不自然 | 102 | 49% | 主体的決断/外部要因/文化/空白等 |

## 3. 帰納的パターン（提案）

### パターンA: 圧力解放型（Pressure-Release）
- **構造:** 圧力(B) → 引き金(F) → 解放(E)
- **該当:** 43件（21%）
- **典型例:** 壬申の乱、応仁の乱、正長の土一揆、桜田門外の変
- **特徴:** 構造的矛盾・圧力が蓄積し、特定の引き金で爆発
- **現行B-F-Eそのもの**

### パターンB: 主体的行為型（Deliberate Action）
- **構造:** 状況(S) → 判断(J) → 行為(A) → 帰結(E)
- **該当:** 58件（28%）— 最大多数
- **典型例:** 大宝律令制定、遣唐使派遣、安土城築城、廃藩置県
- **特徴:** 行為者が状況を認知し、目的を持って行為する
- **B-F-Eとの齟齬:** Bは「圧力」ではなく「状況認知」、Fは「引き金」ではなく「判断」

### パターンC: 外生衝撃型（Exogenous Shock）
- **構造:** 衝撃(X) → 対応(R) → 適応(E)
- **該当:** 21件（10%）
- **典型例:** 仏教伝来、元寇、鉄砲伝来、関東大震災
- **特徴:** 外部から来るもの。内部の「圧力蓄積」は関係ない
- **B-F-Eとの齟齬:** Bは「蓄積された圧」ではなく「受入れ側の状態」

### パターンD: 矛盾露呈型（Contradiction Exposure）
- **構造:** 潜在的矛盾(C) → 顕在化(M) → 構造変化(E)
- **該当:** 42件（20%）
- **典型例:** 墾田永年私財法、承久の乱の帰結、鎌倉幕府滅亡
- **特徴:** 矛盾が「蓄積して爆発」するのではなく「露呈して変わる」
- **B-F-Eとの類似:** 圧力解放に近いが、解放が瞬間的でなく漸進的

### パターンE: 権力空白・移行型（Power Transition）
- **構造:** 空白/死(V) → 後継争い/手続(P) → 新体制(E)
- **該当:** 27件（13%）（権力空白7 + 軍事衝突中の権力争い約20）
- **典型例:** 継体天皇即位、藤原良房摂政就任、徳川家康将軍就任
- **特徴:** 「圧力」ではなく「ポスト」の空きが駆動力
- **B-F-Eとの齟齬:** 解放すべき「圧」が存在しない

### パターンF: 偶発・連鎖型（Contingency-Cascade）
- **構造:** 偶発事象(I) → 連鎖反応(C) → 帰結(E)
- **該当:** 6件（3%）
- **典型例:** 生麦事件、応天門の変
- **特徴:** 偶然の出来事がシステムの脆弱点に触れて連鎖

## 4. 共通構造と可変構造

### すべてのパターンに共通するもの
- **E（状態変化）:** すべてのイベントには何かが変わった帰結がある
- **BGレーン（構造的因子）:** パターンに関係なく、背景にある構造的因子は存在する
- **eGenerates:** E→BGへの影響は普遍的

### パターンごとに異なるもの
- **Bに相当するもの:** 圧力(A) / 状況(B) / 受入れ状態(C) / 矛盾(D) / 空白(E) / 脆弱性(F)
- **Fに相当するもの:** 引き金(A) / 判断(B) / 衝撃(C) / 顕在化契機(D) / 手続(E) / 偶発(F)
- **B→F→Eの因果の向き:** 一方向(A) / 目的的(B) / 外→内(C) / 内的露呈(D) / 構造的(E) / 偶然(F)

## 5. 設計方針（案）

### 案1: 統一名称変更
B/F/Eの名称を「圧力/揺らぎ/解放」から、より汎用的な名称に変更する。

| 現行 | 新名称案 | 意味 |
|------|---------|------|
| B (Background) | **Conditions** | イベント前に存在した条件群 |
| F (Fluctuation) | **Catalyst** | 条件を変化に転じさせた契機 |
| E (Emancipation) | **Outcome** | 観測された状態変化 |

構造は3層のまま、メタファーを変える。「圧力→解放」から「条件→契機→帰結」へ。

### 案2: パターンタグ制
各イベントに `causalPattern` タグを付与し、B/Fの解釈を切り替える。

```json
{
  "eventId": "EV_xxx",
  "causalPattern": "deliberate_action",
  "causalFramework": {
    "E": [...],
    "F": [...],   // deliberate_actionの場合は「判断」と解釈
    "B": [...]    // deliberate_actionの場合は「状況認知」と解釈
  }
}
```

### 案3: 可変中間構造
E（帰結）とBGレーンは固定。その間の構造をパターンごとに変える。

```json
// 圧力解放型
{ "pressure": [...], "trigger": [...], "E": [...] }

// 主体的行為型
{ "situation": [...], "judgment": [...], "action": [...], "E": [...] }

// 外生衝撃型
{ "receiving_state": [...], "shock": [...], "E": [...] }
```
