myHistory 仕様 v3.0（B/F/E 全面移行）

実装確定版 — 2026-02-23

1. 概要

v2.2 の C(Context)/F(Fluctuation)/P(Path)/R(Result) 4レイヤー構造を、B(Background)/F(Fluctuation)/E(Emancipation) 3レイヤーに全面移行する。

  • C → B（Background）: レイヤー名リネーム。カテゴリプレフィックス C- → B-。
  • F → F（Fluctuation）: 変更なし。各イベントで必須1。
  • P → 廃止: Path レイヤーを削除。各FWの notes.legacyPath に旧P要素情報を保存。
  • R → E（Emancipation）: レイヤー名リネーム。カテゴリプレフィックス R- → E-。各イベントで1つ。

8. 検証方法

  • ブラウザで localhost:8080/src/index.html を開き表示確認。
  • 221イベントすべて表示されること。
  • 各行: B要素（複数）→ F要素（1つ）→ E要素（1つ）の順。
  • P / R / C のコードが画面に出ていないこと。
  • 統計パネルでレイヤー B / F / E が正しく集計されること。
  • コンソールエラーなし。
